Added Gateway API support to stateless-svc and restructured all networking config under a single `network` key to eliminate the kustomize post-processing workaround for NetworkPolicies.

- `stateless-svc/templates/routing-ingress.yaml`: replaces `ingress.yaml`; gate changed from `ingress.hosts` to presence of `network.ingress`; mutual-exclusion `fail` guard added
- `stateless-svc/templates/httproute.yaml`: new; renders `gateway.networking.k8s.io/v1 HTTPRoute` when `network.gateway` is present; gateway name defaults to `<release-name>-gateway`
- `stateless-svc/templates/ingress.yaml`: deleted
- `stateless-svc/templates/networkpolicy.yaml`: updated to use `network.inbound`/`network.outbound`; added `fromRouter` rule, `allowDns` flag, `except` support on outbound CIDRs
- `stateless-svc/templates/NOTES.txt`: updated post-install output for both routing types
- `stateless-svc/values.yaml`: replaced `ingress.*` and `egress.*` with `network.{ingress,gateway,inbound,outbound}`; `routing` wrapper removed after review
- `stateless-svc/Chart.yaml`: bumped to 1.6.0
- `stateless-svc/MIGRATION.md`: new; documents all breaking value changes with before/after examples and TLS callout for gateway users
